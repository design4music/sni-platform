## 2. Month Picker Design Pattern

### Option A — Horizontal month strip (recommended)

* 6–12 months visible (scrollable if needed)
* Label: `Jan 2026 · Feb · Mar`
* Active month highlighted
* Lives where the “related countries” box used to be

**Why this works**

* Temporal awareness without cognitive load
* Scales cleanly to years later (pagination / jump-to-year later)
* Zero ambiguity about “what month am I in?”

## 3. Inside a Track (CTM page) — clarify hierarchy

You currently have:

* **Main column**: CTM summary + Topics (events)
* **Right sidebar**:

  * Track switcher (vertical)
  * Some extra menus (optional / removable)

### What to add (important)

#### A. Local Month Switcher (secondary)

* Smaller than centroid-level picker
* Same months, same order
* Changes **only this track’s CTM**

This allows:

* “USA → Economy → Jan vs Feb” without bouncing back

But:

* It must visually subordinate to the centroid-level picker
* Never contradict it silently

## 5. One subtle but important UI rule

### Time never changes silently

* If month changes:

  * Visual indicator must change
  * URL must change
  * Header must say it explicitly

No “why does this look different?” moments.

---

## 6. Mental model summary (this is gold, keep it)

* **Centroid** = stable subject
* **Track** = analytical lens
* **CTM** = monthly publication
* **Month picker** = time lens
* **Archive** = history, not clutter

You’re building **periodicals**, not feeds.
